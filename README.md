> *A maze is not just a puzzle тАФ it's a graph. Every cell is a node. Every corridor is an edge.*

**Maze Intelligence Visualizer** turns abstract computer science theory into a **tangible, playable, cinematic experience**. ## тЬж Algorithms Implemented

<details>
<summary><b>ЁЯЯж Breadth-First Search (BFS)</b></summary>

<br/>

BFS explores the maze in **concentric rings of equal hop-count** from the start cell. By processing nodes layer by layer using a FIFO queue, it guarantees that the first time the exit is reached, the path found is the **shortest possible** in terms of number of moves.

- **Data Structure:** Queue (FIFO) тАФ `Array.shift()`
- **Time Complexity:** O(V + E)
- **Space Complexity:** O(V)
- **Optimality:** тЬЕ Shortest path (unweighted)
- **In-app colour:** ЁЯЯж Blue cells

</details>

<details>
<summary><b>ЁЯЯз Depth-First Search (DFS)</b></summary>

<br/>

DFS plunges as deep as possible along a single branch before backtracking. Implemented with an **explicit LIFO stack** (iterative, not recursive) to avoid JavaScript call-stack limits on large grids. Backtracking events are visually highlighted in the animation.

- **Data Structure:** Stack (LIFO) тАФ `Array.pop()`
- **Time Complexity:** O(V + E)
- **Space Complexity:** O(V)
- **Optimality:** тЭМ Not guaranteed тАФ first path found, not shortest
- **In-app colour:** ЁЯЯз Orange cells

</details>

<details>
<summary><b>ЁЯЯг Dijkstra's Algorithm</b></summary>

<br/>

Dijkstra finds the **minimum-cost path** through weighted terrain. Edge weights (1тАУ9) are deterministically generated per-edge via a coordinate hash, ensuring reproducibility. Uses a custom **binary min-heap** with lazy stale-entry deletion instead of decrease-key.

- **Data Structure:** Min-Heap Priority Queue
- **Time Complexity:** O((V + E) log V)
- **Space Complexity:** O(V)
- **Optimality:** тЬЕ Minimum cost path (weighted)
- **In-app colour:** ЁЯЯг Purple cells

</details>

<details>
<summary><b>ЁЯМА Maze Generation тАФ Recursive Backtracker</b></summary>

<br/>

The labyrinth is procedurally generated using a **randomised DFS** (Recursive Backtracker). Starting from (0,0), the algorithm carves passages into random unvisited neighbours until every cell has been visited, producing a *perfect maze* тАФ fully connected with exactly one path between any two cells. Extra passages (~2.5% of walls) are then removed to introduce cycles for Dijkstra.

</details>
